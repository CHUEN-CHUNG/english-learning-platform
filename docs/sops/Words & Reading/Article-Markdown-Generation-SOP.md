# 文章提取與排版指南 (PDF to Article Markdown SOP)

## 目的
本指南用於指導 AI 或開發者，如何將掃描版或純影像的 PDF 檔案（例如兒童英語教材、故事書），透過 OCR 技術轉換為文字，並重新整理成語意通順、排版良好、段落分明的 Markdown (`.md`) 文章。

## 1. 提取原始文字 (OCR)
- **工具建議**：建議使用 `pytesseract` 搭配 `PyMuPDF` (fitz) 或 `Pillow` 來進行文字提取。
- **原始輸出**：初步提取的文字可能會非常破碎，包含許多斷行與雜訊，這是正常現象。

## 2. ⚠️ 常見資訊碎片化原因與處理對策
AI 在整理原始文字時，**必須**注意並修復以下問題：
- **PDF 的硬換行與排版特性**：將因為頁面寬度限制而換行的句子重新接合，確保完整的句子不被截斷。
- **插圖與多欄位干擾**：忽略或刪除因為插圖排版而產生的雜訊字元（例如圖片旁的標籤或無意義字母）。
- **頁碼與裝飾符號**：主動刪除散落在段落中的頁碼（如 `[20]`, `22`）與排版裝飾線（如 `= (ec`）。
- **特殊字體辨識錯誤**：若遇到標題或特殊字體辨識成亂碼，需根據上下文或常理進行人工/AI 修正推敲。

## 3. 語句重構與清理
- **修正拼字錯誤**：修復明顯的 OCR 辨識錯誤（例如 `T have` 修正為 `I have`）。
- **拼湊斷字**：將被連字號切斷的單字（例如 `holi-day`, `excit-ing`）重新組合。

## 4. 語意分段 (重要)
- **禁止過度細碎**：不要將每一句話都獨立成一行或一段。
- **邏輯分組**：將句子依據故事的發展脈絡（如：背景介紹、事件發生、高潮、結局）分組為 **4 到 8 個邏輯清晰的段落**。
- **對話處理**：角色的對話可以獨立成段，以增加閱讀的流暢度。

## 5. 📝 輸出規範
- **檔案命名**：`{pdf檔名}-Article.md`
- **儲存路徑**：`content/Article/{pdf檔名}-Article.md`
- **格式**：一律輸出為 **Markdown 格式 (`.md`)**。**嚴禁**輸出為 `.txt` 檔。
- **標題**：文章開頭需加上適當的 Markdown 標題（例如 `# The Pirate's Ring`）。
- **段落間距**：段落與段落之間必須保留一個空白行。

## 6. 範例提示詞 (Prompt for AI)
當您需要 AI 處理新的 PDF 時，可以使用以下提示詞：

> 「請參考 `@Article-Markdown-Generation-SOP.md` 的規範，將 `@檔案名稱.pdf` 轉換為分段良好的文章。請務必修復 OCR 碎片化問題、拼湊斷行與斷字，並依據故事邏輯分組為 4 到 8 個段落。最後請加上 Markdown 標題，並輸出為 `檔案名稱-Article.md`。」
