# 文法講義生成規則 (Grammar Handout Generation SOP)

## 🎯 目的 (Goal)
當提供原始的文法學習素材（如 `.pdf`、`.md`、逐字稿或文字敘述）時，AI 需嚴格依照此規則，自動重新排版並生成一份**同時包含「學生填空版」與「教師解答版」**的單一 Markdown (`.md`) 講義檔案。

---

## 📄 講義結構規範 (Structure)

產出的 `.md` 檔案必須嚴格遵守以下由上到下的三大區塊結構：

### 區塊一：【學生填空版】(Student Version)
位於檔案的上半部，供學生上課做筆記與測驗使用。

1. **標題設定**：以 `# 單元名稱` 作為開頭。
2. **填空設計**：
   * 將所有的關鍵文法知識點、公式重點、重要例句的單字等，替換為全形的填空底線 `________`。
   * **中文提示**：為了幫助學生學習，若例句或填空處的句子有中文翻譯，需在填空底線或句子旁加上「括號與中文解釋」，例如：`She is ________. (她在那裡。)` 或 `On（可站立、行走、大空間）: ____________ (搭公車)`。
3. **隱藏防呆內容**：
   * **絕對不可**在學生版中顯示「📌 考試最常錯」或教師專用的教學筆記。
4. **考前檢查表**（若有整理綜合錯誤）：
   * 在學生版的最後一節（例如「考前檢查表」或「綜合複習」），**必須**在該單元標題前加上換頁符號（詳見下述代碼），讓檢查表能獨立成新的一頁。
   * 檢查表內的正確答案一樣要用 `________` 挖空（格式如：`❌ Where you live? → ⭕ Where ________ you live?`）。

### 區塊二：【分頁符號】(Page Break)
在學生版結束、教師版開始之前，**必須**精準插入以下換頁標籤代碼，確保列印成 PDF 時，教師版絕對會從新的一頁開始：

```html
---
<div style="page-break-before: always;"></div>
---
```

### 區塊三：【教師解答版】(Teacher Version)
位於檔案的下半部，供老師備課、上課對答案與提示重點使用。

1. **專屬標題**：加上明確的解答前綴，如 `# 【解答版】文法講義：單元名稱`。
2. **教學小撇步**：在標題正下方，**必須**加上這行斜體提示文字：
   `*(教師小撇步：標記 ==高亮== 的文字為學生版的填空處，可透過尋找取代 ==.*?== 為 ________ 一鍵轉換為學生考卷！)*`
3. **解答高亮標示**：
   * 對應學生版所有被挖成 `________` 的地方，在教師版中都要填入正確答案。
   * **必須使用 Markdown 的高亮語法 `==答案==`** 標示這些填空處（例如：`She is ==there==. (她在那裡。)`），不要使用粗體或其他語法。
   * 請確保中文提示括號一樣保留。
4. **保留防呆筆記 (📌 考試最常錯)**：
   * 在各個文法小節的段落後方，**必須**加入 `📌 考試最常錯` 區塊，列出學生容易犯的錯誤與觀念對比。
5. **考前檢查表解答**：
   * 教師版的最後也需附上完整的考前檢查表，並將所有的錯誤訂正與答案用 `==答案==` 標示清楚（格式如：`❌ Where you live? → ⭕ Where ==do== you live?`）。

## 📝 各單元內容細節規範 (Unit Content & Formatting)

每個文法單元的內容必須結構清晰，並嚴格遵守大小寫與格式規定：

### 1. 每個單元應包含的內容 (Unit Components)
* **大標題/子標題**：清楚標示文法主題（如：`## 3. When (何時) / What time (幾點)`）。
* **文法公式與規則**：例如倒裝句型、介係詞用法等。
* **例句與填空**：提供【問句】與【答句】的情境例句，並在重點處挖空。
* **（僅限教師版）📌 考試最常錯**：彙整學生常見的觀念混淆與陷阱題。

### 2. 大小寫規範 (Capitalization)
* **句首大寫**：每個英文句子的開頭第一個字母**必須**大寫。
* **填空處大小寫**：如果挖空的地方位於句子的最開頭，學生版填空底線的解答、以及教師版的解答（`==答案==`）都必須是**首字母大寫**（例如：`==Where== do you live?` 或 `==It== is raining.`）。
* **專有名詞**：人名（Maria）、地名（Taipei）、星期（Monday）、月份（May）等，無論在哪個位置都必須大寫。

### 3. 格式與排版規範 (Formatting)
* **移除反引號以防變色**：英文例句與對話**絕對不要**使用 Markdown 的行內程式碼反引號（`` ` ``）包覆。直接使用純文字輸入即可，以免在某些閱讀器中文字被渲染成奇怪的顏色（如橘粉色/豬色）。
* **項目符號縮排**：善用 `*` 或 `-` 列表，並保持清晰的縮排結構。
* **字體強調**：使用雙星號 `**粗體**` 來強調文法規則，避免使用斜體以免排版雜亂。

## 💡 AI 執行指令範例 (Prompt Template)

未來請直接使用以下指令（或類似敘述）並附上教材，即可快速產出標準化講義：

> 「這是一份新的文法教材（附檔）。請根據 `@docs/sops/grammar/文法講義規則.md` 的規範，幫我生成一份包含『學生填空版』與『教師解答版』的 `.md` 講義。請確保所有關鍵知識點都已挖空，保留中文翻譯對照，並在解答版提供高亮解答與『📌 考試最常錯』的提示。」
