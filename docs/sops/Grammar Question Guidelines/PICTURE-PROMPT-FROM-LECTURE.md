# 從講義 .md 產生「每單元產圖 prompt」（給 AI 的規格）

之後你只要把講義 Markdown 貼上或 `@檔案`，並說「依規格產生 picture prompt」或「產出 `-picture.md`」，AI 應依本文件產出與 `WHQA-picture.md` 相同骨架的指令集。

---

## 輸出約定

| 項目 | 約定 |
|------|------|
| 輸出檔名 | 與講義同資料夾：`{講義檔名}.md` → `{講義檔名}-picture.md`（例：`WHQA.md` → `WHQA-picture.md`） |
| 檔首 | 可保留一行標題與一句說明（例：「以下每則可單獨複製貼給圖像生成工具」） |
| 單元標題列 | `## 單元 N — …` 或合併時 `## 單元 N、M — …`（與講義章節對照為佳） |

---

## 如何從講義切「單元」

1. 以 **`## ` 開頭**的區塊視為一個單元（一章一則產圖指令），自該行起至下一個 `## ` 之前為範圍。
2. 檔案最上方的單一 **`# ` 全篇標題**不當成產圖單元，僅作講義名稱參考。
3. 若使用者**明確要求合併**（例：「9、10 合併」「11 12 和在一起」），將指定章節合為**一則**指令，`內容細項` 內註明 `（講義第 N、M 單元）`，仍盡量維持**分成三格**（內容過多時可四格並註明理由）。

---

## 每一單元必須使用的指令格式（逐字結構）

以下為**固定骨架**；`{…}` 由講義內容替換。

```markdown
## 單元 {編號或編號範圍} — {短標題}

請為我生成一張圖，要求如下：
主題與來源：參考來源內容細項，製作一張「漫畫學文法：{與該單元主題一致的中文標題}」。
畫面佈局要求（參考三個圖片資源）：
1.請編寫「文法博士」與「新手小白」，博士負責點出規則重點，小白負責說例句。
2. 目前對話偏多，圖片是漫畫風格，字越少越好。
3. 圖片需全英語。
4. 絕對不可列出重複內容。

內容細項：{英文或中英關鍵詞}（{講義檔名}.md 第 {N} 單元），請照下面敘述分成三格
1. 👉 {第一格：句型／規則一句＋例句，例句另起行}
2. 👉 {第二格}
3. {第三格：對照、答句型、易錯點或博士一句短結}
```

### 格式細則

- **不要**在「主題與來源」「畫面佈局要求」等標籤外加 Markdown 粗體，以保持與既有 `WHQA-picture.md` 一致。
- 「主題與來源」固定以 **「參考來源內容細項，製作一張『漫畫學文法：…』。」** 起句。
- 內容細項第二行起：**預設「請照下面敘述分成三格」**；僅在單元內容為並列四點檢核（如四個常犯錯誤）時改為 **「分成四格」**。
- **漫畫畫面內文字**：全英語；**本指令檔**內可保留中文說明與「博士一句」提示。
- 優先從講義中擷取已有例句、**`👉` 句型列**、對錯例（❌／⭕）；不足時再依該節規則補最短例句。

---

## 三格內容怎麼分

1. **第一格**：最常見問句或核心句型 + 1～2 個英文例句（分行）。
2. **第二格**：變形、助動詞句型、或第二組規則 + 例句。
3. **第三格**：答句、介係詞／對照、易錯提醒，或「博士一句對照」（與前兩格區分概念）。

合併多章時：三格可對應「章 A 重點」「章 B 重點」「對照／共同易錯」。

---

## 使用者可下達的簡短指令範例

- 「請依 `Content/grammar/PICTURE-PROMPT-FROM-LECTURE.md`，根據 `@Foo.md` 產出 `Foo-picture.md`。」
- 「講義在 `@Grammar/Bar.md`，每 `##` 一單元，格式同 WHQA-picture。」
