# 英語學習平台 - 產品策略與系統架構 V4 (Product Strategy & Platform Architecture V4)

## 1. 執行摘要與產品願景 (Executive Summary & Product Vision)

本專案旨在打造一個「以數據賦能線下教學，用遊戲化驅動學習動機，將傳統紙本測驗轉化為具備成就感與即時回饋的互動體驗」，本互動涵蓋「閱讀 (Reading)」、「文法 (Grammar)」與「教師後台 (Teacher Zone)」的OMO (Online Merge Offline) 虛實融合的樞紐。
為學生最大化學生的學習吸收率，為家長提供精準的學習成效，更教師加速教材準備時間與教材精確性。

## 2. 目標受眾與 JTBD (Target Audience & Jobs-to-be-Done)

本平台服務三大核心受眾，透過解決其真實痛點來創造產品與商業價值：

### 2.1 學生 (Students) 
*   **JTBD (任務與動機)**：當我在「實體課堂中」進行練習與測驗時，我想要透過遊戲化、無壓力的互動模式來學習，這樣我就能提升單字的記憶成效，在測驗中獲得高分與成就感，進而不再抗拒英文。

### 2.2 教師 (Teachers) 
*   **JTBD (任務與動機)**：當我在進行課堂教學與備課時，我需要一個能自動批改、即時統整學生弱點，並能快速生成/上傳教材的數據後台，這樣我就能大幅減少備課與行政時間，並利用數據精準調整教學策略，避免學生反覆犯錯。

### 2.3 家長 (Parents) 
*   **JTBD (任務與動機)**：當我想要了解並參與孩子的英語學習時，我希望能看到具體、量化的進步數據與課堂活動成果，這樣我就能確認補習班的教學價值，感受到孩子對英文的熱愛，並能更有效地協助規劃他們未來的教育路徑。

*(註：系統開發與維護者為內部利益關係人，不列入產品 TA，其需求與規範詳見下方「系統架構」章節。)*

## 3. 產品 OKR 與核心指標 (Product OKRs & Key Metrics)

由於本平台為線下課程的「必修輔助工具」，產品與學生之間仍需要老師協調。且平台使用場景侷限於**課堂中**，因此 V4 階段的指標將全面聚焦於**「學習成效轉化 (Effectiveness)」、「課堂學習效率 (Efficiency)」與「高階應用能力 (Application)」**。

### Objective 1: 顯著提升學生的單字記憶力與課堂學習效率
*策略意義：透過遊戲化機制，加速短期記憶的建立，提升課堂時間利用率。*
*   **KR 1.1 (記憶轉化)**：「單字總測驗第一部分」的平均正確率，較前置的「同反義詞配對遊戲」提升 **15%** 以上。（驗證遊戲化練習對正式測驗的提分效果）
*   **KR 1.2 (持續進步)**：學生在連續三個單元的「單字總測驗」中，平均成績呈現正成長趨勢（或維持在 85 分以上的高標）。
*   **KR 1.3 (學習效率)**：在閱讀、單字能力皆提升的情況下，每單元教學時間縮減。

### Objective 2: 培養學生將單字與片語靈活應用於實際語境的能力
*策略意義：超越機械式死背，建立真實的語言輸出與應用能力。*
*   **KR 2.1 (應用轉化)**：「單字總測驗第二部分（應用題）」的平均正確率達 **80%** 以上。
*   **KR 2.2 (獨立作答)**：學生在進行「單字總測驗第二部分（應用題）」的第二次重測時，平均錯誤率較首次測驗降低 **20%** 以上。（註：目前對標的「片語重組題」為紙本測驗，現階段先以線上應用題的「重測進步率」作為替代指標。未來將開發「線上拖曳式片語重組模組」以實現全數位化追蹤。）
*   **KR 2.3 (高階輸出 - 未來指標)**：學生在無提示下，完成「單元造句/中翻英」的語意正確率達 **75%** 以上。（註：目前中翻英為紙本作業。現階段可先由教師在後台手動輸入紙本批改成績；未來將導入「AI 語意批改模組」，讓學生直接在線上輸入句子並獲得即時回饋。）

### Objective 3: 賦能教師，打造數據驅動的教學指揮中心
*策略意義：降低教師行政負擔，提升教學決策品質與教材精準度。*
*   **KR 3.1 (備課效率)**：優化內部內容產製流程，使教師單元備課與出題時間平均減少 **30%**。
*   **KR 3.2 (數據依賴度與教學調整)**：**80%** 的活躍教師每週至少登入一次「教師大廳」查看班級錯題分析，並能具體指出依據數據調整了哪些教學重點（例如：針對高頻錯題進行補救教學）。
*   **KR 3.3 (遊戲化教學成效)**：教師於課堂中引導學生完成「互動遊戲（如同反義詞連連看、打地鼠）」後，該班級在對應單元的「單字總測驗」平均成績達 **85 分**以上。（驗證教師運用互動遊戲作為教學輔助，能實質帶動班級整體的成績提升，取代對傳統紙本教材的依賴。）

### Objective 4: 建立家長信任感，將學習成果轉化為品牌口碑與續報動能
*策略意義：將學生的學習成效具象化，讓家長成為教育規劃的合夥人，進而提升商業轉換率。*
*   **KR 4.1 (成果可視化)**：每月/每期發送的「學生學習成果報告（包含遊戲化成就與測驗數據）」，家長開啟與閱讀率達 **70%** 以上。
*   **KR 4.2 (學習態度轉變)**：透過期末家長問卷調查，超過 **80%** 的家長表示「明確感受到孩子對英文學習的抗拒感降低，甚至會主動分享課堂遊戲」。
*   **KR 4.3 (教育規劃與商業轉化)**：家長基於平台提供的學習數據，主動向補習班詢問進階課程、檢定規劃或續報的比例提升 **15%**。（驗證數據作為教育規劃的實質效益）

## 4. 系統架構 (System Architecture)

本專案採用 **Monorepo (多應用模組)** 架構，具備**高度模組化 (High Modularity)** 與 **UI 元件化 (Componentization)** 設計，大幅提升擴充性 (Scalability) 與維護性。
在 V4 版本中，我們進一步將平台入口簡化並重構為「閱讀」、「文法」與「教師專用」三大主軸，並徹底統一了全站的 UI 佈局與元件樣式。

### 4.1 目錄結構 (Directory Structure)
- **`apps/`**: 存放所有獨立的遊戲與應用程式。
  - `reading-hub/`: 閱讀大廳，整合所有閱讀與單字相關的學習資源與測驗。
  - `grammar-hub/`: 文法大廳，整合所有文法相關的遊戲與挑戰。
  - `teacher-hub/`: 教師大廳，整合班級管理、數據追蹤與教材生成工具。
  - 其他獨立遊戲模組（如：同義詞遊戲、打地鼠、火車重組等）。
- **`content/`**: 存放所有的學習教材與題庫（`.csv` 題庫、`.md` 說明文章、`.pdf` 講義）。依據 `vocabulary/`、`grammar/`、`Article/`、`ReadingQA/`、`handouts/` 進行嚴格分類。
- **`shared/`**: **全站共用核心 (Core Shared Modules)**
  - `game-core/`: 存放跨遊戲共用的業務邏輯與數據引擎。
  - `styles/`: 存放 CSS 元件庫 (`hub-layout.css`, `ui-components.scss`) 與全域樣式。
  - `components/`: 存放共用 UI 工廠 (`UIFactory.js`)。
  - `config/`: 存放統一的設計代幣 (`tailwind-theme.js`)。
- **`docs/`**: 存放全域的系統架構文件、產品指南 (`Website-Guide-V4.md`)、開發規範與 SOP。

### 4.2 核心技術與模組系統 (Core Technologies & Modular Systems)

#### A. 統一 UI 系統與設計代幣 (Unified UI & Design Tokens)
- 導入 **Design Tokens** 概念，將語意化顏色統一於 `tailwind-theme.js` 中定義。
- **大廳共用佈局 (Hub Layout)**：所有大廳（閱讀、教師、文法）皆強制使用 `shared/styles/hub-layout.css` 進行佈局與元件樣式統一，並透過 `theme-reading`, `theme-teacher`, `theme-grammar` class 進行主題切換。
- **共用 UI 工廠 (UI Factory)**：透過 `shared/components/UIFactory.js` 動態渲染大廳卡片，確保資料結構與 HTML 結構的一致性。
- **AI 開發規範**：已建立 `.cursor/rules/ui-consistency.mdc`，強制 AI 與開發者遵守 UI 一致性規範，禁止使用 Inline Style 或 Local `<style>`。

#### B. 雙層學習數據追蹤器 (Dual-Layer Data Tracking)
- **`GrammarDataTracker.ts`**: 統一的事件與成效收集器。
- **雙層資料結構 (LocalStorage)**：存放在全站共用的 `grammar_platform_data` 與 `word_exam_all_data` 鍵值中，紀錄共通指標與特定遊戲專屬指標。

#### C. 共用教師儀表板與邏輯渲染 (Unified Teacher Dashboard)
- **`GrammarDashboard.ts`**: 將全站跨遊戲的學習數據進行動態彙整。
- 透過隱藏的快捷操作動態將儀表板 UI 注入畫面，提供總覽數據與錯題分析。

## 5. 核心模組詳細規格 (Core Modules Specifications)

### 5.1 閱讀大廳模組 (Reading Hub)
[📄 詳細規格書：`apps/reading-hub/data/Reading-Hub-Spec.md`]
將原本分散的單字與閱讀功能整合為單一入口，提供結構化的學習路徑：
*   **程度與單元選擇**：支援多種程度（如 YLE Flyers, TOEFL Junior, TOEFL），點擊進入對應單元。
*   **單元學習行動 (Unit Actions)**：
    *   **同反義詞連連看 (Synonyms Game)**：支援下拉選單選擇特定文章段落或全文章進行練習（段落數量由 `units.json` 動態設定）。
    *   **閱讀練習 (Reading Practice)**：支援下拉選單選擇特定文章段落進行閱讀理解（段落數量由 `units.json` 動態設定）。
    *   **課堂講義下載 (Vocabulary Handout)**：提供基於 SOP 自動生成的 PDF 講義下載。
    *   **例句重組測驗下載 (Phrase Reorganization Quiz)**：提供基於 SOP 自動生成的 PDF 測驗卷下載。
    *   **單字總測驗 (Comprehensive Vocabulary Quiz)**：進入該單元的總結性聽寫與拼字測驗。
    *   **課後問卷 (Unit Questionnaire)**：收集學生學習回饋。

### 5.2 文法大廳模組 (Grammar Hub & Games)
[📄 詳細規格書：`docs/architecture/Product/Grammar-Hub-Spec.md`]
*   **文法大廳**：作為所有文法遊戲的唯一入口，學生可先選擇文法主題（時態、基礎文法等），再選擇挑戰模式。
*   **文法遊戲群**：
    *   **打地鼠選擇題 (Multiple Choice Fast)**：結合反應力與文法判斷。
    *   **火車重組題 (Unscramble)**：透過拖曳/點擊車廂重組正確的英文句型結構。
*   **魔法文法海報編輯器**：提供教師編輯並匯出文法教學海報（此功能目前預設隱藏，視需求開啟）。

### 5.3 教師大廳 (Teacher Hub)
[📄 詳細規格書：`apps/teacher-hub/docs/Teacher-Hub-Spec.md`]
建立專屬教師的備課與成效追蹤中心：
*   **安全機制**：進入大廳需輸入密碼（預設 `admin`）。
*   **單元學習數據 (Unit Data)**：點擊按鈕直接進入遊戲的教師數據面板（目前已實作「單字總測驗」的直達功能，其他遊戲的數據追蹤功能尚在開發中）。
*   **成效追蹤**：提供講義與例句重組測驗的下載次數追蹤（已完成，透過 `ProgressTracker.js` 全域紀錄）。
*   **問卷回復**：收集與管理學生的問卷回饋，顯示各單元完成問卷的學生總數（已完成）。
*   **單字遊戲生成器**：提供教師上傳 CSV 與文章，自動生成測驗所需的資料。
