## 0:15–0:55｜學生端 ① · 低壓重試（同反義詞連連看）

**【畫面】** Reading Hub → 進入同反義詞連連看 → 故意配錯一組 → 看回饋 → 再配對成功一組

**【旁白】**

老師可以根據手邊的 PDF 檔匯入教學素材，若 PDF 為圖片，平台也可以自動轉為文字。  
教材匯入後就會產生同反義詞連連看、閱讀大富翁、單字總測驗等遊戲。  
從低難度的配對、選擇題，進階到重組，最後到寫作，讓學生在遊戲的環境中，以低壓、正回饋、不斷重複的方式，一步步將所學完整輸出。  
我觀察到，跟紙本測驗相比，學序漸進的遊戲較可建立孩子的成就感，也比較知道如何在日常生活中應用。

## 0:55–1:45｜學生端 ② · 情境文法

**【畫面】** Grammar Hub → 偵探與記者 → 選一個 case → 只跑完一個 tense（例如 Past：選動詞 → 選時間 → 完成一句）→ 不要四個 tense 全跑  
（或／且）Traveler Quest 第一關示範 — **待決定只秀其一或分段**

**【旁白】**

除了閱讀與寫作，我也另外建立了 Grammar Hub。  
這裡不是只做選擇題，而是把文法放進偵訊對話裡——學生要依時態選動詞、選時間，組出正確問句。  
像我的進階學生人物誌 Ethan，已有文法概念，但需要在真實情境裡反覆用到會。  
這關的錯誤回饋目前是規則驗證，不是 LLM 自由批改，好處是課堂上回饋穩定、可預期。  
Grammar 還有打地鼠、火車重組、Traveler 旅遊三關等模組，這次只示範情境型代表。

第二段是 Grammar Hub 的 Super Traveler Quest，用轉盤選目的地，練 How do you go… 和狀態回覆。  
這是把 Wh- 問句放進旅遊情境，學生要在 NPC 對話裡自己造句，不是只選 ABCD。  
三關分別練交通狀態、How often / How long、購物 How much / How many；這裡示範第一關。  
同類還有偵探對話、打地鼠、火車重組等，模組化後新增題目成本較低。

**【字幕】** 情境文法 · 規則回饋 · 更多模組見 Case Study

## 1:45–2:20｜教師端 · 錯誤熱點

**【畫面】** Teacher Hub → 儀表板 → 指著錯誤熱點 Top 5（或班級成績區）→ 停留 2–3 秒

**【旁白】**

學生練完之後，老師在 Teacher Hub 看班級數據。  
這裡可以看到錯誤熱點，知道全班常錯在哪個文法或題型，下堂要補什麼。  
平台也有單字生成器 和 講義 PDF 產製，目標是減少備課時間、讓教學對準弱點。  
目前資料主要存在 localStorage，適合我現在的小班課堂驗證；規模化會在 Roadmap Phase 3 上雲。
