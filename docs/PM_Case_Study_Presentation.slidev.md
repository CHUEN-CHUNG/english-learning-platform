# AI 英語學習平台產品案例研究

針對台灣英語 **「低年級太淺、高年級太難」** 造成的學習斷層，我打造 **三端一體** 的英語學習平台。

三個能力：
**文法情境學習 × 遊戲化練習 × 學習成果追蹤**

三個價值：
**學生願意練習 × 老師方便教學 × 家長看見進步**

---
layout: default
---

## 1. 問題定義

### 台灣英語教育目前存在三個連鎖問題：

1. **學習斷層**：低年級沒建立紮實基礎 → 高年級跟不上 → 自信崩潰、放棄學習。
2. **教師孤立**：教學工具不夠全面 → 老師輾轉於各平台 → 教學製作需大量時間，且數據難以記錄。
3. **家長黑盒子**：缺乏可量化的進步報告 → 家長只能憑「感覺」決定是否繼續投資。

| 角色 | 核心需求 | 決策動機 |
| :--- | :--- | :--- |
| **學生** | 在真實情境中玩懂文法，錯了能重練、看見進步 | 感覺「用得出來」才願意持續 |
| **教師** | 依錯誤數據加強教學，利用遊戲模板快速產出情境遊戲 | 省下備課與追蹤時間，專注弱點補強 |
| **家長** | 看見可理解的進步數據（含錯題改善趨勢） | 確認投資有效才願意續費 |

**三者互相強化，缺一不可。** 因此，我專注在傳統補習班與現有工具最缺的部分：**用遊戲讓學生願意練、用數據讓老師知道怎麼教、用報表讓家長看見進步**。

## 2. 競品差異化

| 競品 | 優勢 | 缺口 | 本產品切入點 |
| :--- | :--- | :--- | :--- |
| **Duolingo** | 高遊戲化、強習慣養成 | 無教師後台、無個人錯題追蹤 | 老師可控的教材 + 錯題數據 |
| **Kahoot** | 課堂互動即時、學生投入高 | 建題耗時、無法追蹤個人學習歷程 | AI 一鍵生成 + 錯誤熱點儀表板 |
| **Quizlet** | 單字卡成熟、字庫豐富 | 缺乏文法情境遊戲、無寫作輸出 | 情境文法 + 完整學習閉環 |
| **傳統補習班** | 真人互動、家長信任高 | 高成本、無數據、難規模化、無法遠端 | 低成本複製優質教學邏輯 |

**本產品的核心差異：同時服務學生、老師、家長三端，且三端之間的數據互通。**

## 3. 用戶研究 & 人物誌（續）

### 階段一人物誌（六年前 · Concierge MVP）

> **吳志（18 歲 · 進階者）**：需要即時文法糾正、後續可重複練習，以及有深度的議題討論情境。

> **沛姍（14 歲 · 初學者）**：需要貼近日常的內容、友善無壓力的環境，以及即時的支持與指導。

### 階段二人物誌（現在 · 軟體 MVP）

> **Ethan（9 歲 · 進階者）**：有文法基礎，但不會在真實情境中應用；希望透過情境文法任務與短句練習，以遊戲方式反覆練到會用。

> **Lisa（9 歲 · 初學者）**：對文法概念不清楚，漫畫解說加講義，利於理解；對紙本測驗有壓力，遊戲化任務練習時明顯更主動，不怕答錯、願意重試。

**底層洞察：學生不是不想學，而是缺乏「正向的反饋、低負荷的練習、清晰的目標」。**

## 4. 產品迭代：三次決策

| 階段 | 產品形式 | 觀察到的問題 | PM 決策 |
| :--- | :--- | :--- | :--- |
| V1 | 英語影片分享課 | 作業負荷大、課堂互動不足、學生回饋延遲 | 轉向即時互動 |
| V2 | 英語辯論 / 線上口說課 | 願意開口，但糾正後缺乏複習閉環、難規模化 | 轉向遊戲化、可追蹤任務 |
| V3 | 遊戲化英語軟體平台 | 需建立可追蹤學習閉環 | 用 AI 與數據提升效率與成效 |

### Pivot 邏輯：先建立可複習的學習閉環，再擴展到口說

- **發現**：口說課 → 學生願意開口，但糾正後難複習、難規模化
- **轉向**：遊戲化、低負荷練習 → 閱讀／文法／單字閉環
- **策略**：先做好「敢練 · 會複習 · 教得準」，口說留待 Roadmap
- **原因**：可重練、可追蹤、可放大 → 更符合現階段 MVP 與驗證節奏

## 5. 需求對應矩陣

| JTBD / 痛點 | 產品功能 | 成效指標 |
| :--- | :--- | :--- |
| 初學者需在低壓環境建立基礎 | 漫畫式文法講義 × 遊戲化題型練習 × 角色互動與動畫（規劃中） | 任務完成率、第一次答對率 |

## 6. MVP 現況 & 成效驗證

### 已完成

**Reading Hub**

- **平台**：全站登入、單元進度條、單元問卷（Tally，課堂內完成）
- **遊戲**：同反義詞連連看、閱讀大富翁（段落理解＋單元主旨填空）
- **測驗**：單字總測驗三階段（依定義猜字 → 例句克漏字 → 中翻英短句）

**Grammar Hub**

- **遊戲**：打地鼠選擇題、火車句型重組、偵探與記者對話挑戰、Super Traveler Quest — 交通狀態／頻率與時長／購物情境對話練習
- **回饋**：選擇題附題庫解析；短句／糾正為**預設規則驗證**（非 LLM 自由批改）

## 6. MVP 現況 & 成效驗證（續）

**Teacher Hub**

- **儀表板**：閱讀／文法成績、錯誤熱點 Top 5（學習數據以 **localStorage** 為主，尚待雲端化與規模化驗證）
- **題庫／遊戲生成器**：單字題庫生成器（CSV + 文章）；閱讀題庫依 SOP 產製
- **自動化講義**：課堂講義 PDF、例句重組測驗卷；漫畫文法講義（SOP + AI 圖像產製）

**使用情境**：線下教學為主，目前由**我一人授課**，約 **15 位**學生；平台主要在**課堂內**操作，**尚未連結回家練習**。

## 7. 商業假設 & Roadmap

**商業假設**：先以 **Freemium** 讓老師免費使用 AI 教材生成，驗證老師黏著度；再以「**家長成果報告**」作為付費升級誘因，驗證家長付費意願。若驗證成功，再評估 B2B 補習班授權模式。

| Phase | 目標 | 重點 |
| :--- | :--- | :--- |
| Phase 1（現在） | 驗證學習閉環與教師效率 | Reading / Grammar / Teacher Hub（見 MVP 現況） |
| Phase 2（短期） | 全站數據與體驗統一 | 數據平台與 UI 統一、中翻英題型標準化、文法模組化、同反義詞連連看圖片優化、文法主題內容撰寫、文法題型模組整理 |
| Phase 3（短期） | 課後自主練習，家長可視化學習進展 | 跨裝置自主練習、雲端 DB、家長報告、Hub 動態入口 |
| Phase 4（長期） | AI 個人化 | 錯題練習、AI 糾錯、家長報告升級 |
| Phase 5（長期） | 回家練閉環 | 作業連結／提醒、進度激勵、遺忘曲線推播 |
| Phase 6（願景） | 商業化與 B2C 自主學習 | 內建字典、口說、故事化新手引導與吉祥物、付費／B2B 授權 |
