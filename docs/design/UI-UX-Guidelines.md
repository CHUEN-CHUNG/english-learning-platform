# 英文學習平台 UI/UX 設計與前端開發規範

這份文件旨在定義平台內所有應用程式（如：單字測驗、文法遊戲、大廳等）的統一 UI/UX 風格，確保使用者在不同功能間有著一致的操作體驗。

> **💡 新版架構提示 (v2.0)**：
> 平台已導入「統一設計語彙 (Design Tokens)」。未來的顏色與按鈕樣式皆已模組化。
> 若要更改顏色，請**不要**去改 HTML 檔案，請直接修改總部設定檔：`shared/config/tailwind-theme.js`。

---

## 1. 顏色對照表 (Color Map)
平台採用語意化變數 (Semantic Colors) 來管理顏色。當您在 `tailwind-theme.js` 修改以下變數時，全站對應的元件會自動變色：

### 🔵 Primary (主色調)
*預設為 Tailwind 的 Blue (藍色系)*
- **代表意義**：主要互動、一般性確認、平台主視覺。
- **會影響哪些畫面**：
  - 各遊戲的「開始挑戰 / 開始測驗」大按鈕。
  - 單字測驗的「完成第一階段」按鈕。
  - 遊戲結束畫面的「再玩一次」按鈕。
  - 平台首頁與文法大廳的主視覺色塊。
  - 教師資料庫 (Teacher Dashboard) 裡的藍色主題標籤。
- **對應的 CSS 積木**：`.btn-primary`

### 🟢 Success (成功色)
*預設為 Tailwind 的 Green (綠色系)*
- **代表意義**：完成測驗、答對、進入下一關、以及特定主題（如火車遊戲）。
- **會影響哪些畫面**：
  - 遊戲中的「下一題」按鈕。
  - 單字測驗的「完成第二階段並看成績」按鈕。
  - 火車重組遊戲的「開動火車 (送出)」按鈕（因為火車遊戲以綠色為主視覺）。
  - 教師資料庫 (Teacher Dashboard) 裡的綠色主題標籤。
- **對應的 CSS 積木**：`.btn-success`

### 🟡 Warning (警告/注意/特別狀態色)
*預設為 Tailwind 的 Yellow (黃橘色系)*
- **代表意義**：需要引起特別注意的操作、查看錯誤、或是**特定遊戲中的特殊狀態 (如閱讀測驗的活躍句子/金幣、地圖的特殊事件格)**。
- **注意：這是全站統一的橘黃色系，不僅限於文法區，也包含閱讀、單字等所有模組。如果您在閱讀測驗看到橘色的標記或按鈕，這也是統一設計語彙 (Design Tokens) 的一部分。**
- **會影響哪些畫面**：
  - 測驗結果頁的「查看錯題」按鈕。
  - 歷史成績列表裡的「查看此測驗錯題」按鈕。
  - **閱讀測驗 (Reading Practice)** 的目前閱讀中的橘色活躍句子、金幣顯示框。
  - **閱讀測驗 (Reading Practice)** 地圖上的橘色特殊事件格。
- **對應的 CSS 積木**：`.btn-warning`, `.bg-warning` 等

### ⚪ Secondary (次要/返回色)
*預設為 Tailwind 的 Gray (淺灰色系)*
- **代表意義**：返回、取消、關閉、重設等「不會產生重大資料改變」的操作。
- **會影響哪些畫面**：
  - 所有頁面的「🏠 回首頁 / 返回大廳」按鈕。
  - 錯題回顧頁的「回上一頁」按鈕。
  - 教師資料庫的「關閉 / 登出」按鈕。
  - 火車重組遊戲的「重設車廂」按鈕。
- **對應的 CSS 積木**：`.btn-secondary`

## 2. 按鈕元件規範 (Button Components)

為了確保風格統一，所有按鈕的底層都已經寫死在 `shared/styles/ui-components.scss` 中。工程師在寫 HTML 時，只需套用簡短的 Class 名稱。

- **統一特徵**：圓潤可愛的風格（圓角 `rounded-xl`）、粗體字 (`font-bold`)、Hover 過渡動畫 (`transition`)、帶有微幅立體陰影 (`shadow-md` 或 `shadow-sm`)。

### 尺寸修飾符 (Sizes)
工程師可以透過添加以下 Class 來控制按鈕大小：
- `.btn-lg`：大尺寸 (滿寬)，適用於 Start Screen 的巨大開始按鈕 (`w-full py-4 px-8 text-2xl`)。
- `.btn-md`：一般尺寸，適用於多數送出、返回按鈕 (`py-3 px-6 text-lg`)。
- `.btn-sm`：小尺寸，適用於卡片內的小標籤 (`py-2 px-4 text-sm`)。

## 3. 版面與間距 (Layout & Spacing)

### 3.1 響應式網頁設計 (RWD) 與一頁式滿版體驗 (Single-Page / No-Scroll)
為了確保平台在手機、平板與桌機上都有良好的「遊戲體驗」，所有遊戲畫面必須符合**滿版且不須垂直捲動**的原則：
- **鎖定螢幕高度 (100dvh)**：遊戲的外層容器（如 `body` 或 `#app`）應設定為 `h-[100dvh] overflow-hidden`，以避免手機瀏覽器網址列伸縮造成的畫面跳動，並防止使用者在遊玩時意外觸發網頁上下捲動。
- **彈性空間分配 (Flexbox / Grid)**：使用 `flex-1 min-h-0` 來讓中間的遊戲內容（如地洞、車廂區）能根據剩餘的螢幕高度自動縮放。頭部 (`header`) 應設為 `shrink-0` 保持固定高度。
- **內部滾動 (Inner Scrolling)**：如果某個區塊的內容（如說明文字、題目列表）真的太多，應該只讓該區塊具備 `overflow-y-auto`，而不是讓整個網頁滾動。
- **網格與彈性排版**：避免寫死固定的欄數（例如永遠 `repeat(3, 1fr)`）。應善用 CSS Media Queries 或 Tailwind 的響應式前綴。
  - **範例 1 (Grid)**：手機版預設為 2 欄，平板/桌機 (`md:` 或 `@media (min-width: 640px)`) 才改為 3 欄或 4 欄。
  - **範例 2 (Flex)**：善用 `flex-wrap` 讓子元素在空間不足時自動換行，而非擠壓變形。
- **彈出視窗 (Modal)**：確保 Modal 的寬度能隨螢幕縮放（如 `w-full max-w-lg mx-4`），並在手機版預留左右邊距。

### 3.2 遊戲卡片與對話框 (Cards & Modals)
- **背景**：純白 `bg-white`
- **圓角**：大型對話框使用 `rounded-3xl`，一般卡片使用 `rounded-2xl`。
- **陰影**：彈窗使用 `shadow-2xl`，一般卡片使用 `shadow-md`。

### 3.2 Header 狀態列
- 遊戲內的頂部狀態列應使用 `bg-white shadow-sm p-4 rounded-2xl` 等樣式。
- 狀態標籤（如：時間、進度、分數）應使用淺色背景與深色文字的搭配，如：`bg-indigo-100 text-indigo-800 px-4 py-2 rounded-lg font-bold`。

## 4. 互動回饋 (Interaction Feedback)
- **滑鼠游標**：所有可點擊元素（包含客製化的圖示或標題連點觸發器）都應具備 `cursor-pointer`。
- **不可點擊狀態**：按鈕在無法點擊時（如：未選擇單字前無法送出），應加上 `opacity-50 cursor-not-allowed` 樣式。
- **音效與視覺**：答對時顯示綠色、答錯時顯示紅色震動/音效，確保多感官的回饋一致。
