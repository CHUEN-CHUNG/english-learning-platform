1. 發現痛點 (The Problem)

👨‍🏫 教師端：備課成本極高

將實體的 PDF 或 Markdown 講義轉換為線上測驗，需要手動輸入單字、造句、設定選項，耗費大量時間，缺乏自動化出題工具。

🧑‍🎓 學生端：學習缺乏動力

傳統的單字測驗與文法練習枯燥乏味，難以維持專注力，導致學習留存率 (Retention Rate) 低落。

AI 英語教學平台 Case Study 01

2. 解決方案 (The Solution)

打造一個結合 AI 自動化生成 與 前端遊戲化體驗 的整合型教育平台。

⚡ 一鍵生成 (Automation)

教師只需上傳教材，系統即自動解析文本、萃取單字，並產出結構化題庫 (CSV)，將備課時間從數小時縮短至幾秒鐘。

🎮 遊戲化 (Gamification)

導入「打地鼠文法選擇」、「火車文法重組」等多樣化機制。透過時效壓力與即時回饋，將枯燥的測驗變為好玩的挑戰。

🔄 無縫整合 (Integration)

產出的題庫自動串接至前端遊戲引擎，利用 URL 參數即可動態載入不同單元，實現高度擴充性與靈活性。

3. 產品架構與邏輯 (Architecture & User Flow)

採用現代化 Monorepo 結構，前後端分離，確保系統穩定性與可維護性。

(PDF / CSV / Markdown)

➔

學生遊戲端 (Output)

Vite 互動介面
動態載入題庫遊玩

  • 防呆機制設計： 針對非純文字 PDF、檔案過大、格式錯誤等邊界條件 (Edge Cases) 建立完善的例外處理與 UI 提示。
  • 高擴充性： 前端遊戲獨立為多個 App (如 synonyms-game, vocabulary-quiz)，方便未來快速新增新遊戲模組。

4. 為什麼我適合【刷刷庫】與英文庫？

  • 🎮 深刻理解「教育遊戲化」：如同【刷刷庫】的「鴨鴨廚師」，本專案成功實作了多種遊戲機制，我深知如何透過 UI/UX 降低學習阻力，提升產品留存率。
  • 🤝 絕佳的跨部門溝通橋樑：從 0 到 1 獨立完成本專案，使我具備紮實的技術底蘊。我能聽懂工程師的語言，也能將老闆的商業需求轉化為清晰的 PRD 規格文件。
  • 📈 具備擴展國際市場的敏銳度：專案底層支援靈活的資料抽換，這與【刷刷庫】今年擴展日韓市場的目標高度契合，能快速協助產品進行多語系與在地化架構規劃。

身為 PM，我能帶來的價值：

✔️ 數據驅動決策： 追蹤使用者遊玩數據，優化關卡難度與出題邏輯。

✔️ 細節控管： 嚴格把關 UI 互動流暢度與防呆邏輯漏洞。

✔️ 主動提案： 不只執行，更會主動發掘能提升下載量與營收的新功能藍圖。

5. 產品未來藍圖 (Future Roadmap)

如果由我擔任 PM，我對此類教育產品的短中長期規劃：

Phase 1: 基礎留存優化

  • 導入個人化錯題本機制
  • 實作每日登入與連續學習獎勵 (Streak)

Phase 2: 數據與變現

  • 建置學習數據儀表板 (學生/家長端)

  • 導入 A/B 測試優化轉化率

Phase 3: 市場擴張

  • 架構多語系支援 (i18n) 迎戰日韓市場
  • 建立社群分享機制，透過 UGC 降低獲客成本
