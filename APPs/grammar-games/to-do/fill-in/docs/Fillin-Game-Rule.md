# 國中小專屬：填空闖關與對話泡泡填空 (Chat Bubble Fill-in) 遊戲企劃與規格書

## 1. 產品管理與核心目標 (TPM 視角)
### 1.1 遊戲目標
針對國中與國小學童的認知發展與專注力特性，設計一款充滿趣味、低挫折感的「對話式填空遊戲」。透過與虛擬寵物、外國筆友或卡通角色的對話，將枯燥的文法（如 Be 動詞、現在簡單式）與單字練習融入情境中。

### 1.2 目標客群 (TA)
- **國小學童 (7-12歲)**：需要大量視覺刺激、語音輔助與遊戲化獎勵。
- **國中生 (13-15歲)**：準備段考或會考，需要生活化情境來理解基礎文法與句型。

### 1.3 核心指標 (KPIs)
- **完課率 (Completion Rate)**：關卡長度需縮短，確保單次遊玩 3-5 分鐘內能完成。
- **正向回饋率**：孩子主動點擊「再玩一次」的比例。
- **家長滿意度**：提供學習報表讓家長看到孩子答對了哪些文法觀念。

## 2. 遊戲機制與關卡設計 (Game Designer 視角)
### 2.1 核心玩法 (Core Loop)
1. **角色互動引入**：選擇對話對象（例如：魔法貓咪、來自紐約的同齡筆友）。
2. **對話推進與語音**：NPC 傳送一句話（對話泡泡），**並自動播放標準英文發音**。
3. **填空挑戰 (分級設計)**：
   - **幼童/低年級 (拖曳模式)**：畫面上方出現 3 個帶有圖案的「單字積木」，玩家需將正確的積木**拖曳 (Drag & Drop)** 到泡泡的空格中。
   - **中高年級/國中 (點擊/選字模式)**：挖空處顯示中文提示（如：`我 [是] 一個學生`），下方提供 3-4 個單字按鈕供點擊。
   - **挑戰關卡 (字母拼圖)**：給予單字長度底線（如 `_ _ _ _`），下方提供打散的字母讓玩家依序點擊拼出。
4. **即時回饋與獎勵**：
   - 答對：獲得「星星」或「金幣」，角色會發送開心的貼圖。
   - 答錯：不會扣除生命值（降低挫折感），而是由 NPC 溫柔地給出提示（例如：「這個主詞是 He，所以要配哪個 Be 動詞呢？」），並移除一個錯誤選項。

### 2.2 闖關系統與收集要素
- **寵物養成/貼紙簿**：過關獲得的金幣可以買飼料餵魔法貓咪，或是解鎖虛擬貼紙，增加長期遊玩動機。

## 3. 使用者介面與體驗設計 (UI/UX Designer 視角)
### 3.1 視覺風格 (Visual Style)
- **色彩計畫**：高飽和度、明亮溫暖的色調（如馬卡龍色系），避免過於嚴肅的商務感。
- **字體與排版**：
   - 英文使用圓潤、類似手寫的無襯線字體（如 Comic Sans 替代品或 Quicksand），字體必須**夠大 (至少 20px-24px)**。
   - 挖空處 (Blank) 需有明顯的虛線框或凹槽視覺暗示。

### 3.2 互動體驗 (Interaction Design)
- **防呆與容錯**：
   - 拖曳積木時，靠近空格會有「磁吸效應 (Snap-to-grid)」，幫助小手操作不精準的問題。
- **微動畫與音效 (Micro-interactions)**：
   - **答對**：爆發彩色紙片 (Confetti)、歡呼聲音效 (Yay!)，單字積木發出閃耀光芒。
   - **答錯**：積木輕輕彈回原位，發出可愛的「啵」或「噢噢」音效，避免刺耳的錯誤警告聲。
   - **語音按鈕**：每個對話泡泡旁都有一個明顯的「喇叭」圖示，點擊可重複聆聽。

## 4. 技術架構與實作規格 (Tech Lead 視角)
### 4.1 前端架構 (Frontend)
- **框架建議**：React (搭配 `framer-motion` 處理豐富的過場與彈跳動畫) 或 Phaser.js (若需要更強的遊戲物理引擎)。
- **拖曳套件**：使用 `@dnd-kit/core` 或 `react-beautiful-dnd` 實作單字積木的拖曳填空。
- **語音合成 (TTS)**：整合 Web Speech API 或第三方高音質 TTS (如 Google Cloud TTS, ElevenLabs)，確保發音清晰自然。

### 4.2 後端與資料結構 (Backend & Data)
- **資料格式 (JSON)**：需包含語音檔路徑、中文翻譯與兒童友善的提示。
  ```json
  {
    "level_id": "Basic_Be_Verb_01",
    "theme": "My Family",
    "npc_character": "Magic_Cat",
    "dialogues": [
      {
        "id": 1,
        "speaker": "npc",
        "text": "Hello! I am a cat.",
        "audio_url": "/audio/cat_hello.mp3"
      },
      {
        "id": 2,
        "speaker": "player",
        "text": "Hi! He [blank] my brother.",
        "translation": "嗨！他[是]我的哥哥。",
        "answer": "is",
        "options": [
          {"word": "am", "image": null},
          {"word": "are", "image": null},
          {"word": "is", "image": null}
        ],
        "hint_text": "He 是第三人稱單數，要配哪一個好朋友呢？",
        "audio_url": "/audio/player_he_is.mp3"
      }
    ]
  }
  ```
- **進度儲存**：記錄玩家收集的星星與貼紙，並將錯誤頻率高的題目（如：常把 is 填成 are）記錄下來，作為後續「錯題複習關卡」的依據。
