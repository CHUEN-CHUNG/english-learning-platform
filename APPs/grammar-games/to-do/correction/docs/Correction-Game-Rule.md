# 🐛 抓蟲遊戲 (Bug Catching Game) - 遊戲設計與技術規格文件

**文件版本**: v1.0
**負責角色**: 遊戲設計師 / TL / TPM / UIUX 設計師
**遊戲目標**: 透過遊戲化的「抓蟲」機制，訓練玩家快速辨識並修正英文句子中的文法、拼字或語意錯誤。

---

## 🎮 1. 遊戲設計 (Game Design)

### 1.1 核心玩法 (Core Loop)
1. **觀察 (Observe)**: 系統展示一個帶有 1~3 個錯誤（蟲子）的英文句子。
2. **捕捉 (Catch)**: 玩家點擊或框選認為有錯誤的單字/片語。
3. **修正 (Correct)**: 玩家輸入正確的單字，或從彈出的選擇題中挑選正確答案。
4. **回饋 (Feedback)**: 系統給予即時判定，答對獲得分數與特效，答錯扣除生命值並顯示解析。

### 1.2 遊戲規則與機制
*   **生命值系統 (HP)**: 每局遊戲玩家擁有 3 顆愛心（❤️）。抓錯蟲或改錯字扣 1 顆心。歸零則 Game Over。
*   **時間限制 (Time Attack)**: 每題限時 15-30 秒（依難度遞減）。超時視同答錯，扣 1 顆心。
*   **計分與連擊 (Combo)**:
    *   成功抓蟲並修正：+100 分。
    *   連續答對 (Combo)：Combo x 10 的額外加分。
    *   完美通關 (無傷)：額外獎勵 +500 分。
*   **難度分級 (Levels)**:
    *   *Lv.1 (新手)*: 句子短，單一錯誤（如 Be 動詞錯誤 `He are...`）。錯誤單字會有微弱的視覺提示（如稍微抖動）。
    *   *Lv.2 (進階)*: 句子中長，包含時態、介系詞錯誤，無視覺提示。
    *   *Lv.3 (地獄)*: 複合句，多重錯誤（2-3隻蟲），包含語意邏輯錯誤。

## 🎨 2. UI/UX 設計 (UI/UX Design)

### 2.1 視覺風格 (Visual Style)
*   **主題氛圍**: 輕鬆、護眼、帶有科技感或大自然風格（如程式碼除錯的駭客風，或森林抓蟲的可愛風）。
*   **色彩計畫**: 
    *   主色調：護眼深綠色或深藍色 (Background)。
    *   正確回饋：霓虹綠 (Success) + 歡樂的粒子特效。
    *   錯誤回饋：警示紅 (Error) + 畫面輕微震動 (Screen Shake)。

### 2.2 介面佈局 (Layout)
*   **頂部資訊列 (Header)**: 左側顯示目前分數與 Combo 數，中間顯示倒數計時條 (Progress Bar)，右側顯示剩餘愛心 (❤️❤️❤️)。
*   **主遊戲區 (Main Play Area)**: 
    *   置中顯示巨大的英文句子。字體需夠大（推薦使用 Roboto 或 Comic Sans 變體，增加親和力）。
    *   單字之間需有足夠的點擊熱區 (Touch Target)，避免誤觸。
*   **底部操作區 (Footer/Action Area)**: 
    *   點擊單字後，底部彈出「修正面板」(Correction Panel)，提供輸入框 (Input) 或 3 個選項按鈕。
    *   提供「求救提示 (Hint)」按鈕（每局限用 2 次，使用後顯示錯誤單字位置，但該題得分減半）。

### 2.3 互動體驗 (Interaction)
*   **Hover/Touch**: 滑鼠移過單字時，單字會微微放大並有底色 highlight。
*   **抓蟲動畫**: 點擊單字時，如果該字是錯誤的，會跳出一隻小蟲被網子抓住的動畫；如果是正確的字（誤抓），則會顯示「Miss!」並扣心。

## 💻 3. 技術架構 (Technical Leadership - TL)

### 3.1 前端技術棧建議
*   **框架**: React.js 或 Vue 3 (適合快速構建互動式 UI)。
*   **動畫**: Framer Motion 或 GSAP (處理抓蟲特效、Combo 彈跳文字、扣血震動)。
*   **狀態管理**: Zustand 或 Redux Toolkit (管理分數、生命值、計時器、當前題目狀態)。

### 3.2 題庫資料結構 (Data Structure)
建議採用 JSON 格式儲存題庫，便於未來擴充與 AI 自動生成：

```json
{
  "question_id": "bug_001",
  "category": "Be-Verb",
  "difficulty": 1,
  "original_sentence": "She are playing tennis now.",
  "tokens": ["She", "are", "playing", "tennis", "now."],
  "bugs": [
    {
      "target_index": 1,
      "wrong_word": "are",
      "correct_word": "is",
      "options": ["am", "is", "be"], 
      "explanation": "主詞是第三人稱單數 She，且為現在進行式，Be動詞應使用 is。"
    }
  ]
}
```
*技術細節*：前端渲染時，將 `tokens` 陣列 map 出來成為獨立的 `<span>` 元素，並綁定 `onClick` 事件比對 `target_index`。
