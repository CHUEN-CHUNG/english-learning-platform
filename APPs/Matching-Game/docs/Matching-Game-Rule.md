## 1. 遊戲名稱

同義詞反義詞配對遊戲（Matching Game，舊稱 Synonyms Game）。

## 2. 遊戲目標

透過三欄配對，幫助使用者學習英文單字的同義詞（Synonyms）與反義詞（Antonyms），並與閱讀單元文章段落對齊。

## 3. 系統架構與載入邏輯

### 3.1 題庫與單元對應（標準制：`{單元}-Dictionary.csv`）

- **放置位置**：於 `content/vocabulary/` 下依單元建立子資料夾（例如 `content/vocabulary/YLE-3/`），並放入 **`YLE-3-Dictionary.csv`**（命名規則：**`{單元前綴}-Dictionary.csv`**，其中單元前綴與閱讀教材一致，如 `YLE-3`）。
- **自動成庫**：當該路徑新增符合命名之 `*-Dictionary.csv` 時，系統應以此檔為**該單元之正式題庫**，並依本文件 **§5**、**§6** 完成配對遊戲之載入、依 `Para` 分頁（多遊戲頁可同畫面顯示）與導覽行為。
- **相容舊版**：若該單元尚無 Dictionary 檔，可保留對舊版 `.xlsx` 等檔案之載入作為備援（欄位與分頁規則以備援段落為準）。

### 3.2 網址參數、標題與檔案尋址

- **`unit` 參數**：網址帶入單元編號（例如 `?unit=3` 或相容寫法 `?unit=unit3`，實作上應正規化為數字 `3`）。  
  - 系統依 `unit` **自動尋找**對應之 **`YLE-3-Dictionary.csv`（優先）** 及對應閱讀文章（見下）。  
  - 瀏覽器標題與畫面標題應一致設為 **`YLE-3-Matching`**（規則：`YLE-{單元號}-Matching`）。
- **參數與「檔名」**：使用者可視 `unit` 為在網址上選取之單元識別；其對應之題庫實體檔名為 **`YLE-{n}-Dictionary.csv`**，不必在網址重複完整檔名，但文件與對外說明應寫清楚「`unit=3` ↔ `YLE-3-Dictionary.csv`」之對應關係。

### 3.3 閱讀文章（輔助對照與延伸功能）

- **文章路徑**：於 `content/Article/`（或專案約定之單元子路徑）尋找對應 Markdown，例如 `YLE-3-Article.md`、`YLE-3.md`。  
- **用途**：供教學對照、段落語境檢核；**遊戲分頁之權威來源為題庫 CSV 之 `Para` 欄**（見 §5.1），避免與程式內其他「依文章空行推導段落」之邏輯混用時產生兩套真相（§9 說明與實作收斂方向）。

### 3.4 段落選擇器（UI 與選用參數）

- **段落選擇器（`#paragraph-selector`）**：**僅在載入 Dictionary 題庫且成功解析出題目時啟用**。選項為「**全部段落**」以及依該單元題庫 **`Para`（歸頁後之 `gamePage`）** 去重排序後之「**第 N 段**」；變更時導向 `?unit={n}&para={N}` 或清除 `para` 僅保留 `?unit={n}`。舊版（非 Dictionary）題庫則停用此選單並顯示占位說明，避免誤用。
- **畫面標籤**：須在 `<select>` 旁以文字標示 **「段落」**；**遊戲頁**（`Para` 歸頁後之 `gamePage`）僅作為資料與版面分組，**不再**以畫面底部數字鈕切換，改為同畫面顯示多頁字卡（見 §6.2）或透過頂部段落選單／結算彈窗導覽。
- **`?para=` 網址參數（選用）**：與題庫 **`Para` 欄** 之段落序一致（見 §6.1）；與「全冊多頁遊玩」並存時，規格與實作應區隔：**單元**決定載入哪一份 CSV，**段落**篩選該 CSV 內屬於第 N 段的題目。
- **單元切換**：已移除遊戲內的單元選擇器，單元切換統一由「閱讀大廳 (Reading Hub)」控管，透過網址參數 `?unit={n}` 傳入。

### 3.5 UTF-8 BOM 與 Dictionary 解析（維護必讀）

- **現象**：題庫檔存在且格式正確，但遊戲區 **沒有任何單字卡片**；標題可能未更新（初始化腳本中斷）。
- **原因**：`YLE-n-Dictionary.csv` 若以 Excel「UTF-8 CSV」等方式匯出，檔案開頭常帶 **UTF-8 BOM**（位元組 `EF BB BF`，在字串中為 `\uFEFF`）。表頭第一欄會變成「不可見字元 + `Word`」，`Word` 欄位鍵對不到 → 解析結果 **0 筆**。
- **實作要求**：讀入題庫全文後須 **`stripUtf8Bom`**；表頭正規化（`normalizeDictHeader`）時亦須對每個欄位名稱剝除 BOM。詳見 `apps/Matching-Game/src/main.ts` 內 `stripUtf8Bom`／`parseDictionaryCSV`／`initGame`。

## 4. 版面配置

遊戲畫面分為三欄：

- **左欄**：同義詞英文（Synonyms），沒有中文  
- **中欄**：單字本身（Word）英文 + 圖片  
- **右欄**：反義詞英文（Antonyms），沒有中文  

例如：左邊 `writer`、中間 `author`、右邊 `reader`。

- **響應式與置中 (RWD & Centering)**：遊戲畫面整體置中。在手機或較小螢幕（`max-width: 768px`）時，三欄改為垂直排列。  
- **欄位間距**：左右欄與中欄之間距為原先約兩倍，使版面較寬敞。

## 5. 視覺與樣式規格

- **形狀與大小**：欄位標題（Synonyms / Word / Antonyms）為橢圓形標題列（寬 190px、高 47px）。三欄卡片為**相同大小**圓角矩形（寬 160px、高 200px、圓角約 20px）。  
- **字體**：原則上全黑（`#000`）。中間單字與欄位標題為**粗體**；中間單字字級比兩側約大 10%。左右欄為正常粗細。  
- **互動狀態**：  
  - **Hover**：左右單字可略上浮與陰影；中間單字未選取時 hover 僅加強陰影／上浮，**不出現白框**；白框僅在點擊選取後出現。  
  - **選取**：中間單字為 3D 翻面，正面圖+字、背面英文解釋；點擊後翻面並加粗白框。左右被選取時放大顯示。

## 6. 遊戲邏輯與操作模式

### 6.1 分頁權威：`Para` 欄位（一段一遊戲頁）

- **資料來源**：`{單元}-Dictionary.csv` 內之 **`Para`**（或表頭大小寫相容之 `para`）欄，為**分段與遊戲分頁之依據**。  
- **一段一頁**：每一個**段落序號**對應**一個遊戲頁**；該頁僅包含 `Para` 歸屬於該段之題目（不重複把同一題放在多頁，除非產品另定「複數段練習」模式）。  
- **複數段落在同一儲存格**：若 `Para` 為複合值（例如 `"8, 9, 13"`），**歸頁規則**為：以該列數字中的**最小段落序**作為此題所屬之**遊戲頁**（同一題仍只出現在一頁）。若未來改為「多段各一頁」則須另開版本與資料規格。  
- **與舊規之差異**：不再以「每固定 N 題（例如 4 題）強制換頁」作為主流程；若畫面上單頁題數過多，應以**教材分段**或**編輯題庫**調整，而非覆寫 `Para` 語意。

### 6.2 多遊戲頁同畫面（無底部分頁鈕）

- **版面**：屬於不同 `gamePage`（`Para` 歸頁）之題目可**同時**出現在同一捲動畫面中；各字卡以 `data-page`（或等價屬性）標示所屬遊戲頁，供連線與結算邏輯辨識。  
- **互動**：玩家可依任意順序在各遊戲頁字卡間操作，**不再**透過畫面底部數字鈕切換「目前頁」，亦**無**「點擊某頁後鎖住不得換頁」之分頁鎖定行為。  
- **換段／換頁語意**：若要只看某一文章段落之題組，使用頂部 **段落選擇器**（`?para=`）；若要進入下一文章段落之 Matching，使用結算彈窗 **Next Paragraph**（見 §6.6）；行為以程式實作為準（例如無下一段時導回閱讀大廳）。

### 6.3 單元載入與標題（呼應 §3）

- 依 `unit` 自動載入 **`YLE-{n}-Dictionary.csv`** 與對應文章；標題 **`YLE-{n}-Matching`**。

### 6.4 點擊翻面與選取邏輯

- 點擊中間單字：**立即翻面**顯示解釋並加上**粗白框**（同一次點擊生效）。  
- **單一白框**：同一時間僅一張中間卡為選取目標；點另一張時清除其他卡之 `selected`／`flipped` 與計時器。  
- 約 **3 秒**後自動翻回正面，**白框不消失**直至改選其他中間卡。  
- 背面或正面皆可點左右欄進行配對；3 秒內再點同一張已翻面卡可提早翻回正面並維持白框。

### 6.5 空白卡片、打亂與連線計分

- 無同義／反義時顯示「No synonyms」／「No Antonyms」；空白卡可与需空白答案之題目配對。  
- 左右欄順序須打亂，避免與中欄 id 對齊。  
- 配對成功畫綠線並加分（例如每條 +0.5）；錯誤紅線顯示 x、不扣分；錯誤後白框保留可重試。  
- **時間與分數**：仍以「遊戲頁」為單位維護剩餘時間（例如每頁 5 分鐘起算）；`- 30s`／`+ 30s` 調整時應**同步**寫入各頁剩餘時間，避免多頁同畫面時僅一頁被改動。Score 在下一行。時間到或該局完成時開啟結算彈窗（見 §6.6），**不再**以底部 **Leave** 鈕結束。  
- **離開路徑**：標題區右上 **回閱讀大廳** 連結（與閱讀大廳 **`.back-btn`** 同款：`#007bff`、粗體、無膠囊底與圖示）前往 **`apps/reading-hub/index.html`**。  
- **連線穩定性**：錯誤連線清除後需重繪所有正確連線；多頁同畫面時連線篩選須依字卡所屬 **`data-page`**，避免誤判或漏繪。

### 6.6 結算彈窗與段落導覽

- **觸發**：時間到（例如提示「Time's up!」）或該局條件達成（完成配對等，依實作）後顯示結算彈窗。  
- **按鈕（英文文案）**：  
  - **左**：**Next Paragraph** — 導向同一單元之下一文章段落（依 `#paragraph-selector` 選項或 `para` 遞增）；若已無下一段或無法判定，則 fallback 為回到閱讀大廳（與右鈕一致）。  
  - **右**：**Back to Reading Lobby** — 導向 **`apps/reading-hub/index.html`**。  
- **已移除**：彈窗內單一 **Play Again**（若舊文件仍提及應視為過時）、畫面底部 **數字分頁** 與 **Leave**。  
- **樣式**：標題列 **回閱讀大廳**（`#header-exit-link`）、彈窗雙鈕見 `.header-top-row`、`.header-exit-link`、`.modal-actions`、`.btn-modal-secondary` 等（`shared/styles/style.scss`）。

## 7. 實作備註（中間單字翻面與 `style.scss`）

以下為曾發生之 CSS 問題與修正原則，供維護時對照。

- **現象**：點擊中間單字後，滑鼠仍停在卡片上時看似未翻面，移開才翻面。  
- **原因**：`.flipper` 之 `transform` 被 `:hover` 另一規則整組覆寫，導致 `rotateY(180deg)` 被蓋掉。  
- **修正原則**：已選取且已翻面時，hover 須寫成 **`transform: rotateY(180deg) translateY(-5px);`**（或同等保留 `rotateY(180deg)`）；已選取未翻面可只用平移；或 hover 僅調 `box-shadow` 不動 `transform`。  
- **焦點外框**：中欄可用 `outline: none` 避免誤認白框；若需無障礙再以 `:focus-visible` 補低調外框。

相關選擇器見共用樣式中 `#col-words .word-card`、`.flip-container`／`.flipper` 區塊（實際路徑以專案為準）。

## 8. 實作備註（多頁同畫面與 `main.ts`）

- **`activeConnections` 與計分**：儲存連線時應帶上所涉字卡之 **`dataset.page`**（必要時再 fallback 目前邏輯預設頁），與舊版「僅當前頁可點」不同。  
- **`drawAllConnections`**：繪製時**不**再依「當前選中的底部分頁」過濾，須涵蓋畫面上所有已建立之正確連線。  
- **`updatePageVisibility`**：各遊戲頁字卡應一併顯示（例如一律 `display: flex`），與舊版一次只顯示一頁不同。  
- **導覽**：`READING_LOBBY_HREF`、`navigateReadingLobby()`、`navigateNextParagraph()` 與標題列 `#header-exit-link`（文案：**回閱讀大廳**）、彈窗 `#btn-next-paragraph`／`#btn-back-lobby` 對齊 §6.5～§6.6。
