# 單字填空測驗 (Word Fill-in) 測試指南與 AI 執行提示 v1.6

這份文件記錄了「英文單字互動測驗 (Word Exam)」的系統架構、測試流程，以及**常見問題的預防與排除指南**。

> ⚠️ **【重要維護規則】** ⚠️
> **若未來互動中有任何被確認的改動（不論是 UI 調整、邏輯優化或新增功能），皆須同步更新至本文件 `apps/Vocabulary-Quiz/docs/Word-Final-Exam.md`，以保持系統文件的最新狀態。**

**AI 助手請注意：當使用者提供新的單字表並要求進行測驗時，請嚴格遵循下方的【AI 執行指令】與【防呆檢查清單】。**

## 📂 系統架構與檔案位置

### 🔗 各單元測試連結
以下是目前系統中可用的各單元題庫測試連結（點擊即可開啟本地測試）：
- **YLE-1**: [http://localhost:5173/apps/Vocabulary-Quiz/index.html?unit=1](http://localhost:5173/apps/Vocabulary-Quiz/index.html?unit=1)

本測驗系統由 `apps/Vocabulary-Quiz/index.html` 負責執行，並透過 URL 參數 `?unit={單元編號}` 動態讀取對應的題庫檔案。

- **主程式**：`apps/Vocabulary-Quiz/index.html`
- **主畫面白框與 RWD 設計**：
  - 頁面以 `body` 置中。
  - 主容器 `#app` 為白底卡片，最大寬度 `max-w-5xl`。
  - **響應式設計 (RWD)**：系統全面導入 Tailwind CSS 的響應式前綴（例如 `md:`）。在手機等小螢幕上，字體、內距與元件尺寸會自動縮小以適應螢幕（例如 `#app` 內距為 `px-4 py-6`）；在平板與電腦等大螢幕上，則會放大顯示（例如 `#app` 內距為 `px-10 py-12`），提供更清晰的視覺體驗。
  - 歷史成績列表在手機上為單欄顯示，在平板以上尺寸為雙欄顯示 (`grid-cols-1 md:grid-cols-2`)。
- **題庫路徑與命名規則**：系統支援透過 URL 參數 `?unit={單元編號}` 來動態載入題庫。
  - **智慧尋找機制 (Smart Fallback)**：系統會自動在 `content/vocabulary/` 資料夾下尋找標準化題庫檔。
    1. 優先尋找子資料夾內的 `-Dictionary.csv` 檔 (例如 `content/vocabulary/YLE-2/YLE-2-Dictionary.csv`)
    2. 其次尋找子資料夾內的 `-Dictionary.xlsx` 檔
    3. 若無，則尋找相容舊版的 `.xlsx` 或 `.csv` 檔
  - *範例*：若網址為 `?unit=2`，系統會自動依序尋找上述路徑，只要有任何一個存在即可成功載入。

## 🤖 AI 執行指令 (Prompt for AI)

當使用者提供新的單字表（Excel/CSV 檔案或純文字資料），並表示要進行「單字填空測驗」或「測試新單字」時，請 AI 執行以下標準作業流程：

### Step 1: 確認或建立題庫檔案
1. 確認使用者提供的題庫檔案名稱。**此測驗的專屬參數皆應為 `?unit={單元編號}`**（例如：`?unit=1`）。這代表系統會去 `content/vocabulary/` 資料夾中尋找名為 `YLE-1-Dictionary.csv` 的題庫檔。
2. 若使用者僅提供文字資料，請協助將其轉換並儲存為 CSV (`.csv`) 或 Excel (`.xlsx`) 檔案（必須包含 `Word`、`Definition`、`Example-English 2`、`Example-English 3`、`Example-Chinese 3` 等必要欄位），並存放在 `content/vocabulary/{單元名稱}/` 資料夾中（例如 `content/vocabulary/YLE-1/YLE-1-Dictionary.csv`）。

### Step 2: 執行【防呆檢查清單】(重要！)
在啟動伺服器前，AI 必須檢查題庫檔案中的資料是否符合以下規則，以避免前端程式解析錯誤：

*   **⚠️ 詞態變化檢查 (Stemming Check)**：
    *   檢查 `Word` (Base Word) 是否有出現在 `Example-English 2` (例句) 中。
    *   **允許的變化**：前端程式具備「智慧詞態比對」功能，允許例句中使用單字的變化型態（例如：單字是 `Prepare`，例句中可以是 `prepares`、`preparing`、`prepared`）。系統會自動抓取例句中的變化型態顯示在第二階段的單字庫中。
    *   **禁止的錯誤**：如果例句中**完全沒有**該單字或其合理的變化型態，前端將無法挖空該單字。請務必修正例句。
*   **⚠️ 標點符號檢查**：
    *   確保例句中的單字沒有與標點符號「黏在一起」而導致無法辨識（雖然前端 Regex 已盡量處理，但保持例句乾淨是最佳實踐）。
*   **⚠️ 第三階段例句規範 (Example-English 3)**：
    *   僅 `Example-English 3` + `Example-Chinnese 3`（或 `Example-Chinese 3`）**兩欄皆有內容**的單字才會出現在第三階段並計分。
    *   句型須符合 `docs/sops/Words & Reading/Dictionary-Generation.md` 的「第三階段專用句型規範」與 **YLE Flyers 句型菜單**（6～10 詞；須含 `Phrase` 片語；禁止 because / so / when / if、關係子句、分號雙句、or…or、by…ing 巢狀等）。
    *   載入題庫後，請開啟瀏覽器主控台 (F12)，確認無 `[單字測驗·第三階段 QA]` 警告（含詞數、片語、禁止結構）。
    *   若 Dictionary 有 `Phrase` 欄位，第三階段**預設不顯示**片語提示；學生在該題輸入錯誤答案並離開輸入框（blur）後，才會出現一次。
    *   每題會顯示 `Ex3-Pattern` 句型提示（來自 Dictionary 的 `Ex3-Pattern` 欄位）。
    *   完成三階段後會彈出「本次測驗錯誤分析」（常錯單字、片語、句型、文法）。

### Step 3: 啟動本地測試伺服器
請使用 `Shell` 工具在背景啟動伺服器（若尚未啟動）：
```bash
npm run dev
```

### Step 4: 提供本地測試連結與快取提醒
伺服器啟動後，請直接回覆使用者對應的測試連結。**請根據單元編號產生 `?unit=` 參數**，並**務必附上清除快取的提醒**。
格式如下（請將 `{單元編號}` 替換為實際編號，例如 `1`）：

👉 **[http://localhost:5173/apps/Vocabulary-Quiz/index.html?unit={單元編號}](http://localhost:5173/apps/Vocabulary-Quiz/index.html?unit={單元編號})**

> 💡 **溫馨提醒**：如果您剛剛修改了題庫檔案，但網頁上還是顯示舊題目，這是因為「瀏覽器快取 (Browser Cache)」。請在網頁上按下 `Ctrl` + `F5` (Windows) 或 `Cmd` + `Shift` + `R` (Mac) 強制重新整理，或是使用「無痕模式」開啟連結。

## 👨‍🏫 教師專屬資料庫 (Teacher Dashboard)

為了讓資料科學家與行銷專家能深入分析學生行為與漏斗轉換率，系統內建了隱藏的「教師專屬資料庫」。

### 🔑 如何進入資料庫
在測驗首頁，**連續點擊最上方的藍色大標題「單字測驗 Word Exam」 5 次**，系統會跳出密碼輸入框。輸入預設密碼 `admin` 即可進入教師儀表板。

### 📊 紀錄與分析指標
1. **資料儲存架構 (JSON)**：
   - 採用業界標準，所有學生的資料（包含成績、跳出紀錄、個人檔案）皆以 JSON 格式包成一個大字典，並轉成字串統一存入 `localStorage` 的 `word_exam_all_data` 鍵值中。
   - 資料結構嚴格依照「使用者名字」分開儲存，確保資料的獨立性與擴充性。
2. **時間追蹤 (Time-on-Task)**：
   - 紀錄「第一階段總耗時」、「第二階段總耗時」、「第三階段總耗時」。
   - 精細紀錄「每一題的停留時間」（第一、第三階段計算 Input 聚焦時間，第二階段計算點擊空格到填入的時間）。
   - **行銷與教學應用**：
     - **整體平均耗時**：用於評估題庫難度，若平均耗時過長，可能代表題目太難或例句太複雜。
     - **異常耗時偵測**：若某位學生在特定題目停留過久（例如超過 30 秒），但最後答錯或未作答，代表該單字是其學習盲區；若耗時極短（例如 1 秒內）卻全對，可能是作弊或已經非常熟練。
     - **流失率關聯**：分析「中途跳出」的學生，是否因為在某一階段耗時過久而失去耐心。
   - 💡 **備註：底層資料庫 (`word_exam_all_data`) 完整保留了每位學生在每一題的毫秒級耗時 (`s1TimeMs`, `s2TimeMs`, `s3TimeMs`)，未來若需進行進階資料探勘 (Data Mining) 或機器學習預測，皆可隨時撈取原始數據。**
2. **錯誤分類 (Error Typology)**：
   - 系統會在背景自動將錯誤分類為 `Blank` (未作答)、`Typo` (拼字錯誤)、`Wrong_Choice` (語意選錯)、`Sentence_Error` (句子拼寫錯誤)，並在教師資料庫中統計總數。
   - **最常拼錯單字**：系統會自動統計並列出**「最常拼錯的單字 Top 5」**。
3. **行銷漏斗追蹤 (Funnel Tracking)**：
   - 紀錄 `enter_home` (進入首頁)、`click_start` (點擊開始)、`complete_stage1` (完成第一階段)。
   - **整體參與度 (Global Engagement)**：在儀表板最上方直接加總並顯示所有學生的「主動查榜總次數」與「主動檢討錯題總次數」，讓行銷人員一眼看出專案整體的參與熱度。
   - **使用者參與度 (User Engagement)**：紀錄 `click_review_current` (查看當次錯題)、`click_history_score` (點擊歷史成績)、`click_review_history` (查看歷史錯題)，藉此評估學生對學習回饋的重視程度。
   - 透過 `beforeunload` 事件捕捉 `abandonment` (中途跳出)，計算流失率。
4. **使用者活躍度 (Retention & Gamification)**：
   - 自動計算每位使用者的「連續測驗天數 (Streak)」、「歷史總測驗次數」與「上次測驗時間」，為未來的遊戲化行銷提供數據基礎。
   - **個人化參與度追蹤**：精確紀錄「每位使用者」點擊查看錯題（檢討）與歷史成績（查榜）的次數，幫助行銷人員找出高價值、高動機的潛在客戶。

> 🛡️ **【資料安全與備份原則】**
> 未來若需要新增任何追蹤項目或修改資料庫結構，**絕對禁止直接刪除舊有的 `localStorage` 鍵值（如 `removeItem`）**。
> 任何資料結構的升級 (Migration) 都必須採用「向下相容 (Backward Compatible)」或「保留舊資料備份」的策略，確保使用者過去的測驗紀錄、歷史分數與追蹤數據不會因為系統升級而遺失。

## 🎮 測驗操作說明 (供參考)

本測驗包含三個階段與附加功能：

1. **開始畫面與身分驗證（全站統一登入）**：
   - 測驗不再提供獨立的輸入名字與單元選擇畫面。使用者必須先在「閱讀大廳 (Reading Hub)」完成登入。
   - 從閱讀大廳點擊進入測驗時，系統會自動讀取全域身分 (`currentUser`) 與網址參數 (`?unit=X`)，並**直接跳過歡迎畫面，自動開始第一階段測驗**。
   - 畫面的右下角有一個**紫色的圓形懸浮按鈕 (FAB)**（內有圖示與 "History Score" 英文說明），點擊可隨時進入「歷史成績查詢」畫面。**（註：測驗進行中，此按鈕會自動隱藏）**

2. **第一階段（定義填空）**：
   - 系統會列出所有單字的英文定義（Definition），並顯示詞性 (POS) 與同義／反義詞（若有）。
   - 使用者需在文字框中輸入正確的單字（原形）。
   - 系統會自動忽略大小寫與多餘空白。

3. **第二階段（例句填空 - 點擊配對模式）**：
   - 系統會列出所有例句（Example-English 2），並將目標單字挖空（白底紫框）。
   - 畫面上方有「單字庫」（固定於頂部）。**注意：單字庫會自動顯示該單字在例句中的「實際變化型態」（例如 `prepares`），而非原形（`prepare`）。**
   - **操作方式 (雙向支援)**：
     - 方式一：先點擊題目中的空格（變為灰底），再點擊上方單字庫的單字。
     - 方式二：先點擊上方單字庫的單字（出現紫色外框），再點擊題目中的空格。
   - **自動大小寫**：若空格位於句首，填入的單字首字母會自動轉為大寫。
   - **覆蓋與取消**：點擊已填入單字的空格，單字會退回單字庫；若在已選取空格（或單字）的情況下點擊新目標，舊單字會被退回並由新單字覆蓋。

4. **第三階段（中翻英 - 句子拼寫）**：
   - 僅有 `Example-English 3` 與 `Example-Chinnese 3`（或 `Example-Chinese 3`）的單字會計分；其餘顯示「暫無第三階段題目，請跳過」。
   - 例句須符合 Dictionary SOP 之 **YLE Flyers 句型菜單**（見 `Dictionary-Generation.md`）；每句 6～10 詞、只用一種框架、須含 `Phrase` 片語。
   - 題目中文下方會顯示 **📐 句型提示**（`Ex3-Pattern`）與 **⏱️ 時態提示**（`Ex3-Tense`）。
   - 比對答案時允許常見縮寫（如 `we'll` = `we will`）。
   - 若有 `Phrase` 欄位，須**答錯一次**（有輸入且離開該題輸入框後判定為錯誤）才會顯示片語提示。
   - 完成三階段後會彈出**錯誤弱點分析**（常錯單字、片語、句型、文法）。
   - 使用者需手動輸入完整英文句子（YLE Flyers 簡短單句；比對時忽略大小寫、標點與多餘空白）。
   - 題數顯示為「有例句三」的單字數，非全單元總數。

5. **測驗結果**：
   - 完成三階段後會顯示**受測者姓名**、**測驗總耗時**，以及各階段得分與總分。
   - **總分計算**：`第一階段題數 + 第二階段題數 + 第三階段有效題數`（非固定 ×3）。
   - 系統會自動將成績存入瀏覽器的 `localStorage` 中。
   - 畫面底部提供「查看錯題」與「重新測驗」按鈕。

6. **錯題回顧**：
   - 點擊「查看錯題」可檢視答錯的題目、使用者的答案以及正確答案。
   - 錯題畫面分為「第一階段」、「第二階段」與「第三階段」三個分頁，方便使用者專注檢討單一階段的題目。
   - 底部設有「回上一頁」按鈕，會依據您的進入來源，自動返回「測驗結果」或「歷史成績」畫面。

7. **歷史成績查詢 (History Score)**：
   - 點擊畫面右下角的「History Score」懸浮按鈕，可檢視過去最多 50 筆的歷史成績紀錄。
   - 頁面右上角設有下拉選單，可篩選特定使用者的成績。
   - 點擊單次紀錄的「查看此測驗錯題」按鈕，即可回顧該次測驗的錯誤內容。
   - 底部設有「回上一頁」按鈕，可返回測驗結果或首頁。
